## Workflow
1. Define output behavior in one sentence.
2. Split in independent tasks (backend, frontend, context/docs).
3. Assign dependencies and acceptance criteria.
4. Order tasks by risk.
5. Execute and mark completion.

## Examples
- Backend: adjust normalization; Frontend: add UI field; Docs: update examples.

## Quality Bar
- Each task testable semiautonomously.
- No task with unclear expected result.
- Keep scope bounded.

## Resource Strategy
- No extra files unless task requires runbook.
